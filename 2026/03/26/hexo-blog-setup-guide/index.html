Hexo 博客搭建指南

欢迎来到 Hexo 的世界！

Hexo 是一个快速、简洁且强大的 Node.js 博客框架。它能让你轻松创建、管理和部署你的博客。如果你是一名开发者，或者想要一个高度可定制的博客系统，Hexo 是一个绝佳的选择。

本指南将带你一步步搭建一个属于自己的 Hexo 博客，即使你是新手也能轻松上手。

准备工作

在开始之前，请确保你的电脑上安装了以下软件：

  1. Node.js® 和 npm：Hexo 是基于 Node.js 的，你可以在 Node.js 官网 下载并安装 LTS 版本。npm (Node Package Manager) 会随 Node.js 一同安装。
  2. Git：用于版本控制和部署到 GitHub Pages 等平台。

Hexo 目录结构概览

初始化后的 Hexo 项目会包含以下主要目录和文件：

  • _config.yml：站点的主配置文件，全局设置（如站点标题、作者、语言、主题等）。
  • package.json：项目依赖和脚本配置。
  • source/：存放博客内容的地方。
    • _posts/：你的 Markdown 博客文章会放在这里。
    • _drafts/：草稿文章。
  • themes/：存放你安装的主题。
  • package.json：定义了项目依赖和脚本。
  • scaffolds/：文件模板，用于生成文章、页面等。

配置你的站点

站点的主要配置文件是根目录下的 _config.yml。打开它，你会看到很多配置项。对于初学者，至少需要关注以下几项：

  • title：你的博客名称。
  • subtitle：你的博客副标题。
  • description：博客的描述，通常用于 SEO。
  • keywords：博客的关键词。
  • language：博客的语言，例如 en 或 zh-CN。
  • theme：当前使用的主题名称。

主题配置 (Cactus 主题)

你提到了正在使用 cactus 主题。cactus 是一个简洁、美观的主题，非常适合内容展示。

通常，主题的配置会在主题本身的 _config.yml 文件中进行。在你的 Hexo 项目根目录下，找到 themes/cactus/ 文件夹，里面应该有一个 _config.yml 文件。

你可以在这个主题的配置文件中调整：

  • 导航菜单：设置页面的链接，如 “关于”、”分类” 等。
  • 社交链接：添加你的社交媒体链接。
  • 侧边栏小部件：配置作者信息、标签云等。
  • 其他主题特定设置：例如 cactus 主题可能支持代码高亮风格、评论系统集成等。

注意：如果你对主题的 _config.yml 进行修改，并且该主题是你通过 npm install 安装的，那么在主题更新时，你的修改可能会丢失。推荐的做法是：

  1. 克隆主题库：将 themes/cactus/ 目录下的文件复制到你项目根目录下的 themes/ 文件夹中，并手动修改 _config.yml 中的 theme 设置为你的新主题文件夹名（如果复制到新文件夹）。
  2. 使用 hexo-theme-cactus 插件（如果存在）：有些主题可能提供npm包，可以直接安装并配置。

对于初学者，建议先使用主题自带的默认配置，熟悉后再进行调整。

编写你的第一篇文章

文章通常使用 Markdown 格式编写，并放置在 source/_posts/ 目录下。

每篇文章都必须有一个 YAML front-matter（抬头信息），它位于文件开头，由三个连字符 (---) 包裹。Essential front-matter 字段包括：

  • title：文章标题。
  • date：文章发布日期。
  • layout：文章的布局模板（通常是 post）。
  • categories：文章分类。
  • tags：文章标签。

示例：source/_posts/my-first-post.md

总结

恭喜你！你已经成功搭建并配置了一个 Hexo 博客。

  • 核心命令：
    • hexo new <layout> <title>：创建新文章/页面。
    • hexo generate 或 hexo g：生成静态文件。
    • hexo server 或 hexo s：本地预览。
    • hexo deploy 或 hexo d：部署博客。
  • 配置文件：_config.yml（站点）和 themes/<theme_name>/_config.yml（主题）。
