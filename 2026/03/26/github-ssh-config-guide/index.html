GitHub SSH 配置指南

为什么需要 SSH Key？

SSH (Secure Shell) Key 是一种安全的身份验证方式，可以让你在不输入密码的情况下连接到 GitHub 仓库。这不仅提高了安全性，也大大提升了操作效率。

准备工作

在生成 SSH Key 之前，请确保你的电脑上安装了：

  1. Git：如果尚未安装，请从 Git 官网 下载并安装。
  2. SSH 客户端：大多数 macOS 和 Linux 系统自带 SSH 客户端。Windows 用户可以通过 Git Bash 或 OpenSSH 来使用。

生成 SSH Key

  1. 打开终端或 Git Bash：
    在你的电脑上打开命令终端（macOS/Linux）或 Git Bash（Windows）。

  2. 生成 SSH Key 对：
    运行以下命令，其中 your_email@example.com 应替换为你注册 GitHub 的邮箱地址：

    • -t ed25519：指定使用的加密算法，ed25519 是目前推荐的算法，更安全高效。你也可以使用 rsa 算法，例如 ssh-keygen -t rsa -b 4096 -C "your_email@example.com"。
    • -C "your_email@example.com"：为你的 SSH key 添加一个注释，通常是你的邮箱地址，方便识别。

  3. 保存 Key 文件：
    命令执行后，会提示你输入保存 Key 文件的路径和文件名。

    如果你直接按 Enter，它将使用默认路径和文件名（例如 ~/.ssh/id_ed25519.pub 和 ~/.ssh/id_ed25519）。建议使用默认设置，除非你有特殊需求。

  4. 设置密码（可选但推荐）：
    接下来会要求你输入一个 passphrase（密码）。这个密码会用于加密你的私钥，大大提高了安全性。每次使用 SSH Key 时都需要输入此密码（除非你使用 ssh-agent）。强烈建议设置一个强密码。

  2. 添加到 GitHub：

    • 登录你的 GitHub 账户。
    • 点击右上角的个人头像，选择 “Settings”。
    • 在左侧导航栏中，点击 “SSH and GPG keys”。
    • 点击 “New SSH key” 或 “Add SSH key” 按钮。
    • 在 “Title” 字段中，填写一个用于识别此 Key 的名称（例如 “My MacBook Pro”）。
    • 在 “Key” 字段中，粘贴你刚才复制的公钥内容。
    • 点击 “Add SSH key”。
