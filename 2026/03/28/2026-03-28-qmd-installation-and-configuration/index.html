安装并配置 QMD 本地知识库搜索引擎

Fester Zheng
2026-03-28
OpenClaw › 工具 › 知识管理
QMD, 搜索引擎, 本地部署, 知识检索

今天下午完成了 QMD（Quick Markdown Search）的安装与配置，将其作为 OpenClaw 的 memory backend，实现本地化的知识库智能检索。

背景

OpenClaw 默认使用外部 AI 模型进行记忆检索，这会带来 token 消耗和隐私考量。QMD 是一个完全本地运行的混合搜索引擎，结合了：

  • BM25 全文搜索 - 关键词匹配
  • 向量语义搜索 - 语义相似度
  • 重排序（Reranking） - LLM 优化结果
  • 查询扩展 - 自动生成相关查询

核心优势：减少 95% 的 token 使用，所有处理在本地完成，数据不出设备。

安装过程

1. 通过 Git 源码安装

注意：直接 npm install -g qmd 会安装一个占位包，需要通过源码安装才能获得完整功能。

2. 通过bun安装

QMD 需要依赖 bun 运行时，如果你的系统里没有，需要先安装。

如果你使用的是 macOS，可能还需要通过 brew install sqlite 来确保 SQLite 支持向量扩展。

在 Apple Silicon (M5) 上，模型通过 Metal 加速运行。

  • ✅ 配置 OpenClaw memory backend 为 QMD
  • ✅ 创建并索引 memory collection

收获

  1. 本地化知识检索 - 完全脱离外部 AI 依赖，隐私安全
  2. Token 成本优化 - QMD 可减少 95% 的 token 消耗
  3. 混合搜索能力 - 结合 BM25 + 向量搜索 + 重排序，结果更精准
  4. 自动化配置 - OpenClaw 的 memory_search 工具自动使用 QMD 后端
